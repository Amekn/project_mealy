Status: rejected release attempt; retained negative evidence

The annotated `v0.1.1` tag points to protected-main commit
`f291c49f1b6755ec1b298822d85b3a4c45095fce`. Its
[release workflow](https://github.com/Amekn/mealy/actions/runs/30071875207) ran from
2026-07-24T06:18:14Z through 2026-07-24T06:36:40Z and failed before publication.
GitHub has no `v0.1.1` release. No release assets, attestations, public-package
checks, or signed repositories were published by this attempt.

## Failure

Both `linux-x86_64-gnu` and `linux-aarch64-gnu` jobs failed in
`Build reproducible release archive`. The workflow invoked the installed RPM
smoke with:

```text
MEALY_INSTALLED_SMOKE_ROOT=/tmp
```

The package smoke creates its Mealy home below that root and exercises
`mealyctl service install`. Mealy intentionally rejects a persistent systemd
user-service home below `/tmp`, `/var/tmp`, `tmpfs`, or `ramfs`. Both jobs
therefore stopped with:

```text
invalid service installation: the systemd service requires a persistent Mealy home outside /tmp, /var/tmp, tmpfs, and ramfs
```

This was a contradiction in the release test fixture, not a runtime,
architecture, or distribution failure. The temporary path was supplied by
the workflow itself; the production validation behaved as designed.

## Remediation and release handling

The pre-publication RPM and Arch package smokes and every post-publication
Debian, RPM, and Arch package smoke now create a mode-`0700`
container-overlay directory at `/mealy-smoke` and use it as
`MEALY_INSTALLED_SMOKE_ROOT`. The persistent-home validation remains
unchanged. A packaging regression rejects `/tmp` and `/var/tmp` smoke roots
and requires all five release paths to create and use `/mealy-smoke`.

The pushed `v0.1.1` tag is historical public state and must not be moved,
deleted, or reused. A corrected release must use a new semantic version,
repeat the exact-binary release qualification required for that version, and
pass the complete tag workflow through public-package and repository
acceptance before production readiness is claimed.
