## What changed

- A verified interactive install can now continue directly through guided
  provider setup, service activation, health checks, and the first durable
  chat. Running `mealyctl` in a terminal provides the same guided entry point.
- Guided provider choices cover strictly free OpenRouter models, authenticated
  custom OpenAI-compatible endpoints, credentialless loopback servers,
  ChatGPT subscriptions through the official Codex client, and advanced
  OpenAI or Anthropic API configuration. Direct Claude subscription-token
  routing remains unsupported.
- The terminal chat now supports durable continuation, a bounded recent-chat
  picker, live provider/model health, configured limits, exact usage and cost
  reporting, and an explicit provider-management handoff.
- `install-status`, `update`, `update-status`, `repair`, `rollback`,
  `uninstall`, and shell completion expose the installed owner and keep
  archive updates transactional. Failed same-schema archive activation is
  automatically restored through the previous verified package slot.
- Ubuntu, Debian, Fedora, and Arch users gain signed APT, DNF, and Pacman
  repository construction and public trust verification in addition to the
  existing attested rootless archives and native packages.
- Public release acceptance now repeats the complete install-to-first-chat,
  service restart, durable continuation, diagnostics, and uninstall journey
  using only downloaded release assets.
- Persistent user services continue to reject homes on `/tmp`, `/var/tmp`,
  `tmpfs`, and `ramfs`; release package tests now exercise them from an
  explicit persistent container path.
